# Clock-In — 2026-03-11 — 1001-VIDEO AVATAR (Deployment to AWS)

## 1) Context
- **Why now:** We have a live AWS g5.xlarge instance ready. This is a high-performance GPU instance (NVIDIA A10G), perfect for SadTalker and real-time inference.
- **Goal:** Get the full stack running on the cloud with Docker, GPU support, and public access.

## 2) Objective
- **Primary goal:** Successfully deploy the TalkFlow platform to the AWS instance.
- **Success criteria:**
  - [ ] System dependencies (NVIDIA Docker) installed on AWS.
  - [ ] Repository cloned and configured on the server.
  - [ ] SSL/Domain configuration (or at least public IP access).
  - [ ] First video generated successfully on the cloud GPU.

## 3) Deployment Plan
### Phase 1: Server Preparation
1. Connect via SSH.
2. Run `scripts/cloud_setup.sh`.
3. Verify `nvidia-smi` works inside a container.

### Phase 2: Application Launch
1. Clone the repo: `git clone https://github.com/falarcon3398/talkflow-v2.git`.
2. Configure `.env` from `.env.production.example`.
3. Pull/Build images: `docker compose build`.
4. Launch: `docker compose up -d`.

### Phase 3: Validation
1. Access the frontend via AWS Public IP.
2. Test a "Text-to-Video" generation to verify GPU acceleration.

## 4) Deliverables
- Live URL/IP for the application.
- Operational cloud worker.
