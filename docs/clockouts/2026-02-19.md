## How to Run (Local Setup)

### Prerequisites
- **Python 3.10+**: Recommended for backend.
- **Node.js 18+**: Required for the Vite/React frontend.
- **SQLite**: Automatic (no separate DB server needed for local testing).

### Install Steps
```bash
# Backend Setup
cd api_server
pip install -r requirements.txt

# Frontend Setup
cd frontend
npm install
```

### Env Vars Required
- `DATABASE_URL`: `sqlite:///./talkflow.db` (Default)
- `REDIS_URL`: `memory://` (Forced by `celery_app.py` stubs)
- `MINIO_ENDPOINT`: Stubbed in local dev.

### Run Commands
```bash
# Start Backend (Term 1)
cd api_server
python -m app.main

# Start Frontend (Term 2)
cd frontend
npm run dev
```

## Folder-by-Folder Code Walkthrough

### `api_server/`
**Purpose**: FastAPI backend providing generation orchestration and metadata management.
- **`app/models/`**: Responsible for SQLAlchemy schemas.
  - *Key Files*: Added `avatar.py` for persistent user identities.
- **`app/api/v1/endpoints/`**: API Routing.
  - *Key Files*: Added `avatars.py` (CRUD); modified `jobs.py` (List all jobs).
- **`app/core/pipeline/`**: AI Orchestration logic.
  - *Main Flow*: `orchestrator.py` manages the chain of `TTS` -> `Lipsync` -> `Enhance`.
  - *Notes*: All modules here currently utilize **Mock Stubs** to bypass missing AI binaries.
- **`app/core/storage/`**: File persistence layer.
  - *Notes*: `minio_client.py` is in **STUB mode**, writing to local directories instead of S3.

### `frontend/`
**Purpose**: React + Vite + Tailwind CSS v4 UI with Glassmorphism design.
- **`src/`**: Core application logic.
  - *Key Files*: `App.jsx` modified significantly to handle dynamic `avatars` and `jobs` state.
  - *Main Flow*: `App` polls `videoApi.listJobs()` every 10s to update the gallery.
- **`src/api/`**: API service client.
  - *Key Files*: `client.js` updated with `avatarApi` (list/create) and `videoApi.listJobs`.

### `nginx/`
**Purpose**: Local reverse proxy configuration (Internal routing for dev/staging).

## Known Issues / Limitations
- **Mocking**: Video generation is simulated; it copies the source image instead of rendering frames.
- **Concurrency**: Celery is in `task_always_eager` mode, meaning only one job processes at a time per API instance.
