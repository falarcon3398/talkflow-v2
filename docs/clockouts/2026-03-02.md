### 📞 Live Call — Talking Head (WebRTC)
- **Backend Module (`/app/live_call`):**
    - `router.py`: FastAPI routes for avatars and WebRTC Signaling (WebSocket).
    - `agent.py`: Integrated GPT-4o for chat, Whisper for STT, and TTS-1 for real-time speech.
    - `webrtc.py`: PeerConnection management (signaling, track orchestration).
    - `vad.py`: Native integration of `webrtcvad` for speech segmentation.
    - `tracks.py`: Custom tracks for switching between idle loops and talking clips.
    - `workers.py`: Integration with **Wav2Lip** (live lipsync) and **SadTalker** (idle builder).
- **Static UI:** Created a testing interface at `/live` for easy testing without complex React wiring.

### ⚙️ Infrastructure & Dependencies
- **External Storage:** Created `setup_external_storage.sh` to initialize directory structures on the external disk (`_external`).
- **Dependencies:** Updated `requirements.txt` with `aiortc`, `av`, `webrtcvad`, and `httpx`.
- **Main Integration:** Patched `main.py` to auto-mount `/live` and register the `live_call` router.

## 3) Technical Debt & Gotchas
- **Model Weights:** The system assumes models (`wav2lip_gan.pth`, `SadTalker` checkpoints) are manually placed in `_external/models`.
- **Performance:** Live lipsync (Wav2Lip) is GPU-intensive; on local CPU, it may skip frames or lag unless a strong GPU is available via PyTorch.
