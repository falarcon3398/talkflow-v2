# Clock-Out — 2026-03-18 — 1001-VIDEO AVATAR

## 1) Objective Recap

The primary goal today was to address the server daily-reset problem: every time the AWS EC2 instance restarts, the Docker stack fails to come back up automatically, forcing manual rebuilds from scratch. The secondary goal was to establish proper developer documentation (CLAUDE.md and updated README) for the project.

## 2) Key Achievements & Fixes

### Server Auto-Start & Persistent Deployment Fix

- **Root Cause Identified:** The server had no systemd service configured, so after every reboot Docker containers were not started automatically. Additionally, the original `fix_talkflow_server.sh` script used `docker compose build --no-cache` on every run, which triggered a full rebuild of 3x 21.5GB CUDA images — causing long recovery times and frequent errors.

- **Systemd Auto-Start Service Created:** Wrote and deployed `/etc/systemd/system/talkflow-compose.service` that runs `docker compose up -d` automatically after every reboot. The service is enabled via `systemctl enable talkflow-compose`. After reboot, no rebuild is needed — containers start from cached images in seconds.

- **Fixed `docker-compose.yml` GPU Syntax:** The worker service used `gpus: all` which is invalid in Docker Compose v3.8. Fixed to use the correct `deploy.resources.reservations.devices` syntax with NVIDIA driver capabilities.

- **Added `restart: unless-stopped`** to all services (backend, worker, flower, frontend, nginx, postgres, redis, minio) to ensure containers recover automatically from crashes without requiring manual intervention.

- **Removed `--no-cache` from Build:** Changed `docker compose build --no-cache` to `docker compose build` (with cache). Docker layer cache means that after the first build, subsequent builds only rebuild layers that changed — saving 20-40 minutes per reboot cycle.

- **Created `.dockerignore`:** Reduced the Docker build context from 935MB to ~4MB by excluding `.git`, `__pycache__`, `data/`, `docs/`, `_external/avatars/*/talk/`, and macOS `._*` artifact files. This significantly speeds up every future build.

- **Persistent Volume Mounts Confirmed:** All services mount data from `/opt/talkflow/{models,data/uploads,data/processing,data/outputs}` — these survive container rebuilds and server restarts.

### Documentation

- **Updated README.md:** Replaced the original minimal Gradio/HuggingFace README with a comprehensive deployment and operations guide covering architecture, environment variables, health checks, success criteria, common failure modes, and production rules.

- **Created CLAUDE.md:** Wrote a complete developer reference guide covering the full codebase: repo structure, data flows (text-to-video, live call WebRTC), critical behaviors (lipsync lock, solo Celery pool, static fallback detection), tech stack, and deployment rules.

- **Configured GitHub SSH Access:** Generated an ED25519 SSH key (`~/.ssh/id_ed25519_github`) and configured SSH for `git@github.com`. Remote changed from HTTPS to SSH for all future push operations.
